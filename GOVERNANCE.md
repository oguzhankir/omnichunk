# Governance

## Maintainer model

`omnichunk` is maintained by core maintainers listed in `MAINTAINERS.md`.

## Decision process

- Small changes: maintainer review and merge.
- Larger API/behavior changes: design note in PR description + at least one maintainer approval.
- Breaking changes: require migration notes in README/changelog.

## Release process

- Releases are cut from `main` after tests pass.
- Versioning follows semver.
